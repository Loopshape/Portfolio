 Konzeption

CMS/Framework ja/nein?
Um eine Website so zu gestalten, dass diese nur die benötigten Informationen bereitstellt, die bevorzugt gehandhabt werden, dient die Hilfe eines Content-Management-Systems oder eines Hochsprachen-Compiler-Frameworks wie Laravel4 oder CodeIgniter. Überwiegend alle Datensätze können von einen Framework für die Browserausgabe verwaltet und verschiedenen Nutzerrechten zugeordnet werden.

SASS/LESS?
Bei beiden Compilersprachen für CSS macht sich hinsichtlich der Syntax nicht viel her. Der Einsatz solch einer Compilersprache sollte aber immer zu einer guten Websitekonzeption mit bedacht werden.

jQuery2/AngularJS/Bootstrap?
Für viele Ansprüche reicht jQuery2 meistens aus, um eine Website mit allen erdenklichen Interaktionsfeatures auszustatten. Doch auch AngularJS bietet seine Vorteile, wenn es um Verankerung von Informationen im Frontend geht, die aus dynamischen Werten bestehend den DOM weitreichend manipulieren bis auch ergänzen sollen.

Frontend/Backend Handling?
Jedes Framework hat seine besonderen Settings, um Eingabe und Ausgabe für den Benutzer am Bildschirm zu verarbeiten. Je nach Installation eines CMS kann das ganze Template prozedural oder objektorientiert verarbeitet werden.
